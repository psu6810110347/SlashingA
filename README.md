# SlashingA - Hack and Slash Game

## ภาพรวมของโปรแกรม (Program Overview)
เกม SlashingA เป็นเกมแนว Hack and Slash 2D ที่เน้นความรวดเร็วและบรรยากาศที่สวยงาม พัฒนาด้วย Kivy Framework (Python) หน้าที่ของผู้เล่นคือควบคุมตัวละครฮีโร่เพื่อเอาชีวิตรอดจากคลื่นศัตรู (Waves) ที่จะโหดขึ้นเรื่อยๆ 

**จุดเด่นของเกม:**
- **กราฟิกชุด Tiny Swords**: ใช้ภาพสไปรท์คุณภาพสูงที่มีแอนิเมชันครบถ้วน (วิ่ง, นิ่ง, โจมตี)
- **ระบบ BGM ไดนามิก**: เพลงประกอบจะเปลี่ยนไปตามสถานะของเกม (เมนู, ระหว่างเล่น, จบเกม)
- **ระบบ Scaling ที่ท้าทาย**: ศัตรูจะเก่งขึ้นตามเวลาและจำนวนรอบที่ผ่านไป ทำให้เกมไม่น่าเบื่อ
- **ระบบ Hitbox แฟร์ๆ**: การโจมตีคำนวณจากระยะอาวุธและขนาดตัวศัตรูจริง ทำให้ตีบอสตัวใหญ่ได้ง่ายและเป็นธรรมชาติ

---

## การติดตั้งและการรันโปรแกรม (Installation & Execution)
1. **เตรียม Python**: เครื่องของท่านควรติดตั้ง Python 3.10 ขึ้นไป
2. **ติดตั้ง Library**: เกมนี้ใช้ Kivy เป็นหลัก สามารถติดตั้งได้ด้วยชุดคำสั่ง:
   ```bash
   pip install kivy
   ```
3. **การรันเกม**: เริ่มต้นความสนุกด้วยคำสั่ง:
   ```bash
   python main.py
   ```

**การควบคุม (Controls):**
- **W, A, S, D**: เคลื่อนที่ตัวละคร
- **คลิกเมาส์ซ้าย (Left Click)**: โจมตีศัตรู (ทิศทางตามตำแหน่งเมาส์)
- **Tab**: เปิดแผงรายละเอียดศัตรู (Enemy Detail Overlay) เพื่อดูสเตตัสแบบเรียลไทม์
- **1-9**: เลือกสลับดูรายละเอียดของศัตรูแต่ละตัว
- **P**: หยุดเกมชั่วคราว (Pause)
- **ESC**: กลับสู่หน้าเมนูหลัก
- **Fullscreen**: เกมจะเปิดแบบเต็มหน้าจออัตโนมัติเมื่อเริ่มโปรแกรม

## อธิบายการทำงานของโค้ด (Code Architecture)
ตัวโปรแกรมถูกออกแบบมาให้ทำงานแยกส่วนกัน (Modular Design) ดังนี้:

- **`main.py`**: ไฟล์หลักที่รวมทุกอย่างเข้าด้วยกัน จัดการเรื่องหน้าจอ (ScreenManagement), การรับค่าจากผู้เล่น, ระบบเสียง และการวาดแอนิเมชันตัวละครลงบนจอ (Rendering)
- **`game/game_logic.py`**: หัวใจหลักที่คำนวณเหตุการณ์ในเกม ทั้งระบบ Wave, การสุ่มเกิดของศัตรู, การเกิดของ Perk (ลูกแก้วพลัง), และสูตรการคำนวณความเสียหายที่สมดุล
- **`game/enemy.py`**: เก็บโครงสร้างของศัตรูแต่ละประเภท (Knight, Lancer, Archer, Boss) โดยกำหนดค่าพลังและระยะการโจมตีที่แตกต่างกัน
- **`game/player.py`**: เก็บสถานะของผู้เล่น และระบบการเลเวลอัปที่ค่อยเป็นค่อยไป
- **`ui/widgets.py`**: จัดการ UI ทั้งหมดที่เห็นบนจอ ตั้งแต่แถบพลังชีวิตบอส จนถึงตัวเลขความเสียหายที่เด้งขึ้นมา
- **`events/callbacks.py`**: ตัวกลางที่คอยรับสัญญาณเหตุการณ์ต่างๆ (เช่น ศัตรูตาย, ผู้เล่นเก็บไอเทม) เพื่อประสานงานระหว่างส่วนแสดงผลและส่วนตรรกะ

## Game Design Document (GDD) - ฉบับปรับปรุง

### สถานะผู้เล่น (Player Stats)
- **ระยะอาวุธ (Attack Range)**: 100 px
- **เลเวลอัป**: เพิ่ม Max HP +7, พลังโจมตี +2 และเพิ่มพลังป้องกัน +1 ทุกๆ 2 เลเวล
- **ระบบสมดุล**: ใช้ EXP แบบทวีคูณ (30% ต่อเลเวล) ทำให้เลเวลไม่พุ่งไวเกินไป

### ประเภทศัตรูและการปรับสเกล (Enemy Scaling)
ความยากจะเพิ่มขึ้นทุกๆ **120 วินาที** และเพิ่มความโหดตามจำนวน **Wave**

1. **Knight (สายสมดุล)**: ศัตรูมาตรฐานที่พบได้บ่อย
2. **Lancer (สายรถถัง)**: เลือดเยอะ เดินช้า แต่พลังชีวิตสูงมาก (ปรับลดระยะจู่โจมเหลือ 75 เพื่อให้ผู้เล่นรักษาระยะห่างได้)
3. **Archer (พลธนู)**: โจมตีจากระยะไกล (300 px) ด้วยลูกธนู
4. **Boss (บอสใหญ่)**: เกิดทุกๆ 10 Wave มีเลือดมหาศาล (500+) และ Hitbox ขนาดใหญ่ (220 px) ทำให้การสู้บอสรู้สึกยิ่งใหญ่และกดดัน

### ระบบ Perk (ลูกแก้วพลัง)
- **การเกิด**: สุ่มตำแหน่งทุกๆ **15 วินาที**
- **เงื่อนไข**: เกิดได้สูงสุดเพียง **2 กล่องต่อ 1 Wave** เพื่อความสมดุลของทรัพยากร
- **ผลลัพธ์**: สุ่มเพิ่มค่าพลังโจมตี, พลังป้องกัน, ความเร็ว หรือเลือดสูงสุด
